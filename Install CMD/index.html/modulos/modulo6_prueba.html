📌 Caso práctico: Prueba Técnica

Simulación real de análisis y gestión comercial

🏢 Contexto del ejercicio

Usted es el responsable comercial de la zona Costa Atlántica para la cuenta Relojes en Statur Interprice S.A.S.

  • Venta mayorista
  • Recaudo de cartera
  • Proyección de pedidos
  • Análisis de inventario
  • Planeación mensual
  • Defensa de cifras ante gerencia

Comisión: 2% sobre el recaudo efectivo del mes.

🎯 Objetivo de la prueba

Esta prueba no evalúa solo habilidades técnicas. Evalúa tu capacidad de gestionar una zona comercial completa:

  • Vender
  • Cobrar
  • Analizar
  • Priorizar
  • Planear estratégicamente

📦 Entregables

1. Archivo Excel (.xlsx)

  • DATOS_LIMPIOS
  • ANALISIS
  • CARTERA
  • ABC
  • PLANMES
  • DASHBOARD

2. Informe ejecutivo

Máximo 2 páginas (PDF o Word)

📅 Condiciones del ejercicio

  • Histórico: Enero – Marzo 2026
  • Cartera: corte 31-mar-2026
  • Planeación: Abril 2026

⚠ Importante:

Los recaudos posteriores a marzo (abril) están incluidos solo como referencia de comportamiento, pero NO deben sumarse al recaudo efectivo del primer trimestre.

⚙️ Proceso obligatorio

  1. Crear carpeta de trabajo: PRUEBA_STATUR_FASE1_APELLIDO_NOMBRE
  2. Crear archivos CSV separados
  3. Importar con Power Query o método equivalente
  4. Estandarizar tipos de datos
  5. Generar archivo final en Excel (.xlsx)

📁 Archivos requeridos

  • clientes.csv
  • inventario.csv
  • ventas_ultimos_3_meses.csv
  • recaudo.csv
  • cartera_actual.csv

📌 Reglas clave

  • No se permite pegar datos manualmente en una sola hoja
  • Debe existir proceso reproducible
  • Tipos de datos deben estar correctamente definidos
  • El archivo debe recalcular correctamente en Excel

🧠 Competencias evaluadas

  • Limpieza y estructuración de datos
  • Automatización (ETL)
  • Análisis comercial
  • Planeación estratégica
  • Comunicación de resultados

🚀 Metodología de desarrollo (Fases del proyecto)

A continuación se describe el proceso completo que debe seguirse para desarrollar la prueba técnica de forma profesional, reproducible y alineada con las exigencias de la empresa.

📁 Fase 1: Preparación del entorno

Tarea local obligatoria:

  • Crear la carpeta: PRUEBA_STATUR_FASE1_APELLIDO_NOMBRE
  • Crear los archivos CSV:
  • clientes.csv
  • inventario.csv
  • ventas_ultimos_3_meses.csv
  • recaudo.csv
  • cartera_actual.csv

Los datos deben copiarse exactamente sin modificar nombres de columnas.

⚙️ Fase 2: Motor ETL con Power Query

🔌 Conexión dinámica

  • Conectar los 5 archivos CSV desde la carpeta
  • Parametrizar la ruta para reutilización en otros equipos

🧹 Limpieza y tipado

  • Convertir fechas correctamente
  • Estandarizar textos
  • Convertir valores a moneda (COP)

⚠️ Regla crítica: Recaudo Q1

Solo se deben considerar recaudos de Enero a Marzo con estado "Aplicado" para el cálculo de la comisión.

  • Filtrar Estado = "Aplicado"
  • Excluir recaudos de abril del cálculo
  • Usar abril solo como referencia de comportamiento de pago

📤 Carga

Los datos transformados deben cargarse en la hoja: DATOS_LIMPIOS

📊 Fase 3: Análisis y modelado

📈 ANALISIS

Evaluar rotación de productos vs inventario disponible.

💰 CARTERA

Construir aging de cartera al 31-mar-2026 para identificar riesgos.

📊 ABC

Clasificar clientes según volumen de ventas y recaudo (Regla de Pareto 80/20).

📅 PLANMES

Definir estrategia comercial de abril:

  • A quién vender (ABC)
  • A quién cobrar (Cartera)

📈 Fase 4: Dashboard

Diseñar un panel visual claro y ejecutivo.

  • % cumplimiento de recaudo
  • Comisión proyectada (2%)
  • Top clientes (ABC)
  • Estado de cartera por edades
  • Top productos vs inventario

Regla: evitar saturación visual.

💼 Fase 5: Informe ejecutivo

Documento de máximo 2 páginas con enfoque gerencial.

  • Resumen de ventas
  • Recaudo total
  • Clientes en riesgo
  • Plan de acción para abril

🧠 Reglas avanzadas (Nivel experto)

✔ Regla de oro del recaudo

Solo incluir valores donde Estado = "Aplicado" para cálculo de comisión.

🌍 Compatibilidad regional

Configurar delimitadores y formatos en Power Query para evitar errores en diferentes configuraciones de Excel.

🔍 Trazabilidad total

No se permite copiar y pegar datos manualmente. Todo debe provenir de procesos automatizados.

🔗 Ruta dinámica

Implementar una celda donde el usuario pueda cambiar la ruta de los archivos y actualizar todo el modelo automáticamente.
