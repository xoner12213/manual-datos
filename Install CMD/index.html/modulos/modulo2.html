Módulo 2: Construcción del Modelo de Datos

De datos crudos a información confiable

🧠 Nodo 1: ¿Por qué datos limpios?

Los datos limpios no son un requisito técnico, son una condición estratégica para tomar decisiones correctas.

  • Eliminación de inconsistencias: evitar errores manuales
  • Integridad referencial: coincidencia entre ClienteID y SKU
  • Estandarización: datos homogéneos para análisis

📁 Nodo 2: Origen de la información

El modelo se construye a partir de múltiples fuentes:

  • clientes.csv
  • cartera_actual.csv
  • recaudo.csv
  • inventario.csv
  • ventas_ultimos_3_meses.csv

Todos los datos deben ser importados mediante Power Query o un proceso reproducible.

🏗 Nodo 3: Arquitectura del modelo

🔑 Identificadores (Dimensiones)

  • ClienteID
  • SKU
  • OrdenID

💰 Métricas financieras

  • PrecioUnit
  • ValorOrden (Unidades × PrecioUnit)
  • ValorRecaudo
  • Comisión (2% del recaudo)

📊 Segmentación

  • Ciudad
  • Canal
  • Segmento
  • Formalidad

⚙️ Nodo 4: Transformaciones en Power Query

🧹 Limpieza y normalización

  • Texto limpio
  • Conversión a mayúsculas
  • Eliminación de inconsistencias

📊 Lógica de riesgo

Clasificación basada en días de mora:

  • Mora leve: < 30 días
  • Riesgo medio: 31 - 60 días
  • Riesgo alto: > 60 días

🔢 Tipificación técnica

  • Fechas → tipo fecha
  • Valores → moneda (COP)

🎯 Resultado esperado

Un modelo de datos limpio, estructurado y listo para alimentar el análisis, garantizando trazabilidad y confiabilidad en la toma de decisiones.
